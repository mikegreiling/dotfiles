# Feature demo — replayable Playwright recording

This self-contained Playwright project records a short demo video of a B-Stock
frontend feature, driving the **real** dev backend with a natural-looking cursor.
It was generated by the `bstock-demo-video` skill; everything feature-specific
lives in **`demo.config.ts`**.

## What it does
`npx playwright test` mints a buyer session (no Cloudflare Turnstile — server-side
FusionAuth flow), forces any required LaunchDarkly flags on, hides dev-tool
overlays, then runs each scenario in `demo.config.ts`, recording one `.webm` per
scenario. `npm run encode` trims + transcodes them to `.mp4` (+ a combined clip).

## External dependencies (you must provide these)
1. **Node** ≥ 20 (the recorder itself). Generating the videos originally used the
   portal's Node (24) — any modern LTS runs Playwright fine.
2. **VPN** access to the B-Stock dev network (the backend is VPN-gated).
3. **A running app at `baseUrl`** that actually contains the feature:
   - If the branch is **merged/deployed**, point `demo.config.ts` `baseUrl` at the
     deployed env (e.g. `https://bstock-dev.com`) — nothing to run locally.
   - If **not yet merged**, run that branch's dev server locally (e.g. home-portal
     `npm run dev` → `http://localhost:3030`) and keep `baseUrl` on localhost.
4. **Login credentials** (never committed): a dev test account's **email +
   password**. Not stored in `demo.config.ts` — supply them via env vars **or** a
   git-ignored `creds.json`, whichever you prefer:
   ```sh
   export BSTOCK_DEMO_EMAIL='…' BSTOCK_DEMO_PASSWORD='…'      # option A
   echo '{"email":"…","password":"…"}' > creds.json          # option B (git-ignored)
   ```
   Use any B-Stock dev test account you have that can see the demo's resources
   (and has a destination address if the feature needs one). These are throwaway
   dev-env accounts.
5. **Subject record IDs** in `demo.config.ts` `records` (listings/orders/etc.).
   Auction listings expire — refresh with `npm run discover` and paste fresh IDs.
6. **ffmpeg** (for `npm run encode`): `brew install ffmpeg`.

## Run
```sh
npm ci
npx playwright install chromium
# ensure VPN + the app at baseUrl + credentials (env or creds.json) are set
npm test          # records .webm per scenario into test-results/
npm run encode    # → videos/*.mp4 (+ combined.mp4)
```
Stale record IDs? `npm run discover` lists active candidates to drop into `demo.config.ts`.

## Notes
- Demos **stop before the final Confirm** by default (no mutation of shared dev
  data). A scenario may set `submit: true` to perform the real action — only do
  that on a disposable/seeded record.
- Recording size (viewport == video size) is fixed in `demo.config.ts` `size`.
